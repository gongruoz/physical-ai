# 用手机当摄像头 / 背着手机跑

当前项目是电脑用 USB 连 Arduino，串口发指令控制舵机。想「用数据线连在手机上、背着手机跑、借用手机摄像头」有几种做法：

---

## 方案一：手机当遥控 + 摄像头（不改硬件，推荐）

**接线**：Arduino 仍然用 **USB 连电脑**（电脑可以放背包或桌上），手机和电脑在 **同一 WiFi**。

**步骤**：

1. 电脑上运行：`npm run gesture`
2. 终端会打印本机 HTTPS 地址，例如：`https://192.168.1.100:3750`
3. **手机浏览器必须用 HTTPS 打开**（用 HTTP 时浏览器会禁用摄像头）：`https://192.168.1.100:3750`
4. 首次打开会提示「不安全」→ 点「高级」→「继续访问」或「接受风险并继续」（自签名证书，仅本机局域网使用）
5. 点「开启摄像头与手势」→ 会用 **手机摄像头**（手机端默认用后摄，适合绑在小车上朝前看）
6. 把手机绑在小车上，小车背着手机跑；控制指令路径：**手机浏览器 → WiFi → 电脑 → USB → Arduino**

这样已经实现：**用手机摄像头 + 手机跟着车跑**，只是 Arduino 的线还是连在电脑上。

- 要用后摄：访问 `http://<本机IP>:3750?camera=back`（手机默认已是后摄）
- 要用前摄：访问 `http://<本机IP>:3750?camera=front`

---

## 方案二：数据线连手机（仅 Android，手机当主机）

要让 **Arduino 用数据线连在手机上**、完全不用电脑，只能由 **手机** 跑程序、走 USB 串口。

- **Android**：支持 USB OTG。理论上可以：
  1. 用 USB 线把 Arduino 接到手机（需 OTG 转接线/口）
  2. 在手机上用 **Termux** 装 Node.js，跑 `node gesture-servo-server.js`
  3. 手机浏览器打开 `http://localhost:3750`，用手机摄像头做手势，串口由本机 Node 发到 Arduino

**注意**：Termux 下 `serialport` 对 USB 串口的支持因机型/内核而异，需要自己实测。若 USB 无法识别 Arduino，可考虑方案三。

- **iPhone**：系统不开放 USB 串口给普通 App，**无法** 用数据线直接连 Arduino 当主机。若要用 iPhone，只能走方案一（手机当遥控+摄像头，电脑接 Arduino）或方案三（无线）。

---

## 方案三：改硬件，完全无线（真正「只连手机」）

如果希望 **小车只带手机、没有电脑、也没有从 Arduino 到电脑的线**：

- 把 Arduino 换成（或加）**带 WiFi/蓝牙** 的板子（如 **ESP32**），让小车通过 WiFi 或蓝牙收指令。
- 手机跑一个 PWA 或 App：用手机摄像头 + 手势/逻辑，通过 **WiFi 或蓝牙** 把「角度、速度」发给 ESP32，ESP32 再控制舵机。

这样就是「手机 + 小车」两个设备，中间无数据线，真正背着手机跑并借用手机摄像头。需要改固件和一点电路，不在当前仓库内。

---

## 小结

| 目标                     | 做法 |
|--------------------------|------|
| 用手机摄像头、手机绑车上 | 方案一：电脑跑服务，手机浏览器打开 `http://<电脑IP>:3750` |
| 数据线连手机、手机当主机 | 仅 Android 可试（Termux + USB OTG），串口需实测 |
| 完全不用电脑、不接电脑线 | 方案三：换/加 ESP32 等，手机通过 WiFi/蓝牙发指令 |

当前代码已支持：**方案一**（LAN 监听 + 手机端用后摄）。直接 `npm run gesture`，用手机连电脑 IP 即可。
